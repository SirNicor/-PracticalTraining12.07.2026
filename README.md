# -PracticalTraining12.07.2026
Вариант 38. Составление расписания занятий 
Предметная область: Группы, преподаватели, аудитории, дисциплины, пары. Программная реализация: БД (группы, преподаватели, аудитории, занятия);  функции: составление, проверка накладок, просмотр. 
Рекомендуемый стек: Java + SQLite/PostgreSQL 
Ход работы: 
1) описать предметную область, границы системы и роли пользователей; 2) составить требования и построить ER-модель данных; 
3) реализовать базу данных и функции ввода, редактирования, удаления и поиска  (CRUD); 
4) реализовать проверку накладок (аудитория/преподаватель) и просмотр по группе  и протестировать на 15–20 записях. 
На защите показать: как выявляются конфликты в расписании.
